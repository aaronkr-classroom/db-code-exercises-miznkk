2. 다음 중 DB의 정의에 함축된 의미가 잘못 설명된 것은? c
   - a. DB는 컴퓨터가 접근할 수 있는 저장 장치에 기록된 저장 데이터다. 
   - b. DB는 조직을 운영히는 데 반드시 펼요한운영 데이터다. 
   - c. DB는 한 조직의 여러 부서에서 시용하는 데이터를 중복이 전혀 없는 상태로 한곳에 모아서 공동으로관리히는통합데이터다. 
   - d. DB는 한 조직의 여러 응용 프로그램이 공동으로 사용하는 공용 데이터다. 
3.  디음 중 DB의 특징이 잘못 설명된 것은? d
   - a. DBMS에 의한 관리: DB를 효율적으로 구축하고 관리하기 위해서 DBMS라는 전용 소프트웨어가필요하다. 
   - b. 실시간 접근성: DB는 컴퓨터가 접근할 수 있는 저장 장치에 저장되어 관리되므로 언제든지 필요한 시점에 접근할 수 있다. 
   - c. 지속적인 변화:DB는 끊임없이 변하고 있는 현실 세계를 반영해야 하므로， 그 내용은 고정된 것이 아니고 삽입， 삭제， 갱신 등을 통해서 계속 변화한다. 
   - d. 위치에 의한 참조:DB에 저장된 데이터는 저장된 주소， 즉 위치에 의해 참조되므로 사용자의 요구가빠르게 처리될 수 있다.

   - c. 어떤 통아리가 길거리에서 일회성 특별 이벤트를 개최한 디음， 이 이벤트에 참가해서 이름을 남긴 사람들의 명단을 관리하려고 한다. 
   - d. 매년 조류독감이 발생한 지역과 피해 및 보상 내역을 관리하려고 한다.
6. 3단계 DB 구조가 무엇인지 설명하시오. 
   : DB 구조는 외부 단계, 개념 단계, 내부 단계로 나뉜다. 외부 단계는 개별 사용자 관점의 DB 구조를 정의하고, 개념 단계는 조직 전체 관점에서 DB의 논리적 구조를 정의한다. 그리고 내부 단계는 데이터의 물리적 저장 구조를 정의한다.

   - b. 외부 스키마는 외부 단계에서 특정 사용자 또는 응용 프로그램이 필요로 히는 DB 구조를 정의한것이다. 
   - c. 개념 스키마는 개념 단계에서 범기관적인 관점의 전체 DB를 정의한 것이다. 
   - d. 내부 스키마는 내부 단계에서 개념 스키마의 저장 구조를 정의한 것이다.
